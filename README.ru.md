# Экосистема проектов Nargan

[Read in English (README.md)](README.md)

Единая, отказоустойчивая и динамически оркеструемая веб-экосистема, объединяющая независимые веб-приложения (плагины). Разработанный на базе FastAPI, Python и MongoDB, этот монорепозиторий функционирует как хаб приложений, спроектированный для легкого масштабирования, автоматического обнаружения новых модулей и мягкой деградации в случае проблем с базой данных или системными зависимостями.

---

## Обзор архитектуры

Экосистема работает на основе модульной монолитной архитектуры под управлением центрального координатора:

1. **Динамическое обнаружение плагинов (`main.py`):** При запуске корневой координатор сканирует локальные директории, автоматически ищет валидные роутеры внутри папки каждого проекта (проверяя несколько резервных точек входа, таких как `main.py`, `[имя_плагина].py` или прямые импорты роутера) и монтирует статические файлы и пути API под изолированными префиксами.
2. **Устойчивый статус системы (`/api/status`):** Главная страница динамически опрашивает этот эндпоинт. Если какой-либо плагин не может импортироваться, загрузить свои зависимости или подтвердить точки входа, он помечается как `offline` и автоматически затемняется в интерфейсе пользователя, позволяя остальной части экосистемы работать без сбоев.
3. **Оптимизированный RAM-First трекинг (`/api/track`):** Централизованная аналитика отслеживает уникальных посетителей в MongoDB. Запросы к базе данных минимизируются за счет локального кэша `LRUSet`, что защищает базу данных во время резких скачков трафика.
4. **Двойной запуск (Dual-Boot):** Все основные плагины содержат независимый механизм запуска (`main.py` внутри своих папок), позволяющий разработчикам запускать их по отдельности как десктопное приложение с помощью Eel или как автономное веб-приложение с флагом `--web`.

## Каталог экосистемы

### 🚀 Главный хаб
* **Лендинг (`index.html`, `main.py`):** Пользовательская панель управления, поддерживающая локализацию на разные языки, отображение счетчика посетителей, динамическое скрытие оффлайн-проектов и воспроизведение случайных фоновых видео.

### 📁 Реестр приложений

#### 1. [Formular](./formular/) (Кузница документов и медиа)
* **Описание:** Универсальный конвертер файлов «все-во-все», использующий программный алгоритм поиска кратчайшего пути Дейкстры.
* **Интеграции:** Headless Playwright (рендеринг HTML в PDF), PyMuPDF, LibreOffice, FFmpeg, Pandoc, Pandas, Pillow, CairoSVG и 7-Zip.
* **Возможности:** Связывает разрозненные категории медиафайлов (например, Markdown ➔ HTML ➔ PDF или Excel ➔ CSV ➔ JSON ➔ XML), динамически собирая цепочку выполнения для любого поддерживаемого перехода.

#### 2. [Yellow Mirror](./yellow_mirror/) (Удаленный headless-браузер)
* **Описание:** Трансляция браузера в реальном времени через WebSockets в обход ограничений на стороне клиента.
* **Возможности:** Запускает изолированные сессии Chromium внутри серверного Playwright. Использует протоколы CDP Screencast, сжимая кадры рендеринга в base64 JPEG и передавая их по высокоскоростным веб-сокетам прямо на HTML5 Canvas. Передает движения мыши, клики и мультиязычный ввод с клавиатуры.

#### 3. [Toadcode](./toadcode/) (Среда совместной разработки)
* **Описание:** Интерактивное рабочее пространство с виртуальной файловой системой (VFS) для временных проектов.
* **Возможности:** Поддержка древовидной структуры папок, множественной загрузки файлов, распаковки архивов `.ZIP` и импорта репозиториев с GitHub через прокси. Включает инструмент выделения области (lasso), отображение номеров строк, базовое автодополнение и ограничение размера проекта в реальном времени (10 МБ).

#### 4. [Markbin](./markbin/) (Markdown-редактор и хостинг документов)
* **Описание:** Платформа для создания, просмотра и публикации Markdown-документов на базе движка Vditor.
* **Возможности:** Интерактивное визуальное редактирование, автогенерация оглавления, скачивание файлов на клиентской стороне и ссылки с автоудалением. Использует встроенные TTL-индексы MongoDB для автоматического удаления документов по истечении заданного времени жизни.

#### 5. [Kanban](./kanban/) (Минималистичный органайзер задач)
* **Описание:** Легковесный менеджер задач с рекурсивными вложенными списками и поддержкой drag-and-drop.
* **Возможности:** Бесконечная вложенность задач, экспорт и импорт конфигурации в формате JSON через native file picker, поддержка сенсорных экранов (drag-drop-touch), горячие клавиши и цветовое кодирование проектов.

#### 6. [D&D Tools](./dnd/) (Инструменты Мастера Подземелий)
* **Описание:** Полезные утилиты для Dungeons & Dragons 5th Edition (правила 2024 года).
* **Возможности:**
  * **Бестиарий и помощник по Wild Shape:** База данных с поиском и фильтрацией, поддерживающая поиск по синонимам, исключающие теги (`-`), переключение языков и сложную многофакторную сортировку.
  * **Просмотрщик чарлистов Foundry VTT:** Импорт персонажей из файлов `.json` для наглядного отображения листов персонажа. Парсит формулы бросков кубиков и динамические ссылки в тексте.

#### 7. [Evenfest](./evenfest/) (Сайт сообщества косплееров)
* **Описание:** Шаблонный сайт с управлением структурой напрямую через MongoDB, использующий шаблоны Jinja2 для динамического вывода новостей, списков фотографов, билетов и правил фестиваля.

#### 8. [Snake](./snake/) (Органическая аркада)
* **Описание:** Аркадная игра «Змейка» на Canvas с физикой векторных частиц, динамическим масштабированием сложности и API для подгрузки случайных видеофонов.

## Установка и запуск

Репозиторий настроен для бесшовного запуска на Hugging Face Spaces (с использованием автоматической синхронизации через GitHub Actions), Vercel или локальных серверах.

### Системные требования
Для работы всех функций конвертации и стриминга на хост-системе должны быть установлены:
* Python 3.10+
* MongoDB
* LibreOffice (Headless CLI)
* FFmpeg & FFprobe
* 7-Zip (`7z`)
* Зависимости библиотеки CairoSVG

### Установка

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Nergan/projects.git
   cd projects
   ```

2. Запустите скрипт сборки для автоматической установки зависимостей ядра и каждого плагина:
   ```bash
   chmod +x build.sh
   ./build.sh
   ```

3. Настройте конфигурационные переменные в файле `.env` в корне проекта:
   ```env
   MONGODB_URI=mongodb+srv://<username>:<password>@<cluster>.mongodb.net/
   ```

4. Запустите веб-сервер:
   ```bash
   chmod +x start.sh
   ./start.sh
   ```

Скрипт запустит Uvicorn на порту `7860`, принудительно инициализируя стандартный цикл событий `asyncio` для обхода распространенных проблем с таймаутом SSL-рукопожатия, возникающих на альтернативных loop-раннерах (таких как uvloop).
